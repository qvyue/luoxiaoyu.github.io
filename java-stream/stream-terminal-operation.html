Stream终止操作

通过终止管道操作之后，Stream流将会结束，最后可能会执行某些逻辑处理，或者是按照要求返回某些执行后的结果数据。

collect 返回集合​

collect()方法为最常用的Stream终止操作，目的是返回List、Set、Map等集合类型对象，还能做一些组合操作：

toList/toSet、toMap​

配合collect方法，生成List、Set集合对象

public void testCollectToListSetMap() {
        List<String> stringList = Arrays.asList("Hello", "World", "Tutorial", "Lxiaoyu", "Com");
        List<Integer> toList = stringList.stream().map(String::length).collect(Collectors.toList());
        System.out.println("toList = " + toList);
        Set<Integer> toSet = stringList.stream().map(String::length).collect(Collectors.toSet());
        System.out.println("toSet = " + toSet);
        // 注意toMap的第三个参数【(v1, v2) -> v1】，如果省略的话会报错：map中存在形同的key，因为Hello和World长度都为5。
        // 第三个参数的意思是如果两个key的值相同，则选择第一个
        Map<Integer, String> toMap = stringList.stream()
                .collect(Collectors.toMap(String::length, String::toLowerCase, (v1, v2) -> v1));
        System.out.println("toMap = " + toMap);
}

可以看到toList是正常按序显示，toSet结果中的元素是去重而且乱序的，toMap要注意重复key问题，执行结果如下：

toList = [5, 5, 8, 7, 3]
toSet = [3, 5, 7, 8]
toMap = {3=com, 5=hello, 7=lxiaoyu, 8=tutorial}

joining 拼接流中的元素​

在collect方法中传入joining进行元素拼接，和String自带的join()方法等效，不同点在于拼接字符串之前，stream流可以做复杂的操作，更加灵活。

public void testCollectJoining() {
        List<String> stringList = Arrays.asList("Tutorial", "Lxiaoyu", "Com");
        String result = stringList.stream().map(String::toLowerCase)
                .collect(Collectors.joining("."));
        System.out.println("拼接流中的元素: " + result);
}

执行结果如下：

拼接流中的元素: tutorial.lxiaoyu.com

groupingBy 分组​

在collect方法中传入groupingBy进行分组，其中groupingBy的方法参数为分类函数。

public void testCollectGroupingBy() {
    List<String> stringList = Arrays.asList("Hello", "World", "Tutorial", "Lxiaoyu", "Com");
    Map<Integer, List<String>> groupingBy = stringList.stream().collect(Collectors.groupingBy(String::length));
    System.out.println("groupingBy = " + groupingBy);
}

groupingBy分组类似mysql里面的group by语法，还可以通过嵌套使用groupingBy进行多级分类。执行结果如下：

toMap = {3=[Com], 5=[Hello, World], 7=[Lxiaoyu], 8=[Tutorial]}

partitioningBy 分区​

分区是特殊的分组，它分类依据是true和false，所以返回的结果最多可以分为两组。

public void testCollectPartitioningBy() {
    List<Integer> integerList = Arrays.asList(1, 2, 3, 4, 5);
    Map<Boolean, List<Integer>> result = integerList.stream().collect(partitioningBy(i -> i < 3));
    System.out.println("partitioningBy: " + result);
}

partitioningBy一般应用场景是通过判断条件分为两组相反的组合，然后分别获取再进行下一步操作。执行结果如下：

partitioningBy: {false=[3, 4, 5], true=[1, 2]}

allMatch/anyMatch/noneMatch 匹配元素​

返回一个boolean值，类似于isContains()，用于判断是否有符合条件的元素，具体分为：

  • allMatch 匹配所有元素
  • anyMatch 匹配其中一个
  • noneMatch 全部不匹配
public void testAllAnyNoneMatch() {
    List<Integer> integerList = Arrays.asList(1, 2, 3, 4, 5, 6);
    boolean allMatch = integerList.stream().allMatch(i -> i > 3);
    System.out.println("是否全部匹配：" + allMatch);
    boolean anyMatch = integerList.stream().anyMatch(i -> i > 3);        
    System.out.println("是否任意匹配：" + anyMatch);
    boolean noneMatch = integerList.stream().noneMatch(i -> i > 3);
    System.out.println("是否全部都不匹配：" + noneMatch);
}

执行结果如下：

是否全部匹配：false
是否任意匹配：true
是否全部都不匹配：false

count 统计流中元素个数​

public void testCount() {
    List<Integer> integerList = Arrays.asList(1, 2, 3, 4, 5, 6);
    long count = integerList.stream().count();
    System.out.println("count=" + count);
}

执行结果如下：

count=6

min/max 获取最小最大值​

public void testMinMax() {
List<Integer> integerList = Arrays.asList(6, 2, 3, 4, 1, 5);
    Integer min = integerList.stream().min(Integer::compareTo).orElse(0);
    System.out.println("min = " + min);
    Integer max = integerList.stream().max(Integer::compareTo).orElse(0);
    System.out.println("max = " + max);
}

整数集合求最大最小值，orElse(0)方法表示设置默认值0，执行结果如下：

min = 1
max = 6

findFirst/findAny 查找第一个/任意元素​

findFirst()是查找集合中第一个符合条件的元素，findAny()是查找集合中任意一个符合条件的元素。findFirst/findAny和allMatch/anyMatch方法类似，只是返回值不同，还有findFirst/findAny不传参，要提前筛选符合条件的元素。

public void testFindFirstFindAny() {
    List<Integer> integerList = Arrays.asList(1, 2, 3, 4, 5, 6);
    Integer findFirst = integerList.stream().filter(i -> i > 1).findFirst().orElse(0);
    System.out.println("findFirst = " + findFirst);
    Integer findAny = integerList.stream().filter(i -> i > 1).findAny().orElse(0);
    System.out.println("findAny = " + findAny);
}

相同筛选条件下，貌似findFirst和findAny都一样，但是这是在普通流stream()情况下，如果在并行流parallelStream()情况下，findAny的结果可能每次都不同。执行结果如下：

findFirst = 2
findAny = 2

foreach 遍历​

foreach()方法等效于Java的foreach循环方法，比较常用，在前面的教程中也多次提到

public void testForeach() {
    List<Integer> integerList = Arrays.asList(1, 2, 3, 4, 5, 6);
    integerList.stream().foreach(System.out::println);
}

执行结果如下：

1
2
3
4
5
6

reduce 将流中的元素组合​

reduce() 方法主要用来进行算术运算

public void testReduce() {
    List<Integer> integerList = Arrays.asList(1, 2, 3, 4);
    Integer reduce = integerList.stream().reduce(0, Integer::sum);
    System.out.println(reduce);
}

上面reduce方法是使用Integer的sum方法（Lambdas表达式写法）求和，如果结果不存在则默认结果为0，执行结果如下：

10
